# Laboratório de Experimentação de Software

## Introdução

Este trabalho tem como objetivo analisar características de repositórios populares
do GitHub, considerando os 1000 repositórios com maior número de estrelas.

Antes da coleta e análise dos dados, foram formuladas hipóteses informais para cada
questão de pesquisa.

## Hipóteses

### RQ01 – Sistemas populares são maduros/antigos?

Hipótese: Espera-se que repositórios populares sejam relativamente antigos,
pois projetos mais antigos tiveram mais tempo para acumular estrelas.

### RQ02 – Sistemas populares recebem muita contribuição externa?

Hipótese: Espera-se que projetos populares recebam muitas contribuições externas,
resultando em um alto número de pull requests aceitas.

### RQ03 – Sistemas populares lançam releases com frequência?

Hipótese: Projetos populares tendem a lançar releases com frequência para manter
seus usuários atualizados e corrigir problemas rapidamente.

### RQ04 – Sistemas populares são atualizados com frequência?

Hipótese: Espera-se que repositórios populares tenham atualizações recentes,
indicando manutenção ativa.

### RQ05 – Sistemas populares são escritos nas linguagens mais populares?

Hipótese: Acredita-se que linguagens como JavaScript, Python e Java sejam
predominantes entre os repositórios mais populares.

### RQ06 – Sistemas populares possuem alto percentual de issues fechadas?

Hipótese: Espera-se que projetos populares apresentem uma alta proporção
de issues fechadas, indicando boa manutenção do projeto.
